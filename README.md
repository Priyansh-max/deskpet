A lightweight Windows desktop companion that reacts to your system load. A small
animated pet sits on your desktop and speeds up as your CPU gets busier вЂ” idle CPU
means a slow stroll, heavy load (compiling, gaming, rendering) makes it zoom.

Inspired by the macOS app *Zoomies*. Understand your PC activity at a glance,
without opening Task Manager.

## Features (V1)

- Transparent, frameless, always-on-top pet window in the bottom-right corner
- Animation speed scales smoothly with CPU load (3вЂ“18 FPS)
- Two pets: **Cat** and **Dog**, switchable from the tray
- Pause / resume animation from the tray
- Pet choice persists across restarts
- Runs from the system tray вЂ” no taskbar clutter

## Tech Stack

- **Electron** + **electron-vite** (build tooling)
- **React** + **TypeScript** (renderer)
- **systeminformation** (CPU monitoring)
- **electron-store** (persistence)

## Getting Started

```bash
npm install        # install dependencies
npm run gen:assets # generate placeholder sprite frames + tray icon
npm run dev        # run in development with hot reload
```

## Build & Package

```bash
npm run build      # compile main, preload, and renderer into out/
npm run package    # build an unpacked Windows app into release/
npm run dist       # build a Windows NSIS installer into release/
```

## How It Works

```
systeminformation (main)  в”Ђв”ЂCPU%в”Ђв”Ђв–¶  IPC  в”Ђв”Ђв–¶  React renderer
        every 1000ms                            fps = 3 + (cpu/100)*15
                                                 clamped to [3, 18]
```

The main process samples CPU load once per second and pushes it to the renderer
over a context-isolated IPC bridge. The renderer maps that value to a frame rate
and advances the sprite animation accordingly.

## Replacing the Art

The included sprites are **placeholders** generated by
`scripts/generate-assets.cjs`. To use real art, drop transparent PNG frames into
`src/renderer/assets/<pet>/` named `<pet>_1.png` вЂ¦ `<pet>_8.png` (8 frames). The
loader picks them up automatically by filename вЂ” no code changes needed.
